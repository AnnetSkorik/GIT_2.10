2.Как создать git репозиторий из локального каталога на своем компьютере. Как его отправить на github.com?
Для создания нового репозитория используется команда git init . Команду git init выполняют только один раз для первоначальной настройки нового репозитория. Выполнение команды приведет к созданию нового подкаталога . git в вашем рабочем каталоге.

4.Какую информацию нам дают команда :
git status
git log
git remote -v 

Команда git status отображает состояние рабочего каталога и раздела проиндексированных файлов. С ее помощью можно проверить индексацию изменений и увидеть файлы, которые не отслеживаются Git. Информация об истории коммитов проекта не отображается при выводе данных о состоянии. Для этого используется команда git log .
Команда git remote используется, чтобы указать, на каких удаленных конечных точках будут работать команды синхронизации.

5.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?

Это команда git clean — удобная утилита для отмены изменений в рабочем каталоге, и команда git reset , которую можно вызвать с параметрами --mixed или --hard , чтобы сбросить изменения в рабочем каталоге.

6.Как вернуть файл из индекса в рабочий каталог?

Это позволяет сделать команда git rm , которая также удаляет файл из рабочего каталога.

7.Как отменить ошибочный commit ?

Удалить последний коммит можно очень просто с помощью команды git reset .

8.Как исправить коммит отправленный на сервер GitHub?

git rebase -i и указываем ей коммит, который мы хотим изменить.
